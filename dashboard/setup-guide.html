TubeStack setup guide

One-time Google Cloud configuration for the YouTube Data API v3 key and OAuth (playlist tools), plus a short note on OpenAI for optional AI features in the dashboard.

Google Cloud — one project

TubeStack uses a single Google Cloud project for both the public YouTube Data API v3 key (metadata, channel scans) and an OAuth 2.0 Web client (sign-in for creating playlists on YouTube and importing your playlists). You can complete everything in the Google Cloud Console.

1. Create or pick a project

  1. Open Google Cloud Console and select Create project (or an existing project you use only for TubeStack).
  2. Give it a clear name (for example “TubeStack”) and create it, then select that project in the top bar.

2. Enable YouTube Data API v3

  1. Go to APIs & Services → Library.
  2. Search for YouTube Data API v3 and open it.
  3. Click Enable.

3. Create an API key (Data API)

  1. Go to APIs & Services → Credentials.
  2. Create credentials → API key.
  3. Optional but recommended: under Application restrictions choose what fits how you use Chrome; under API restrictions, restrict the key to YouTube Data API v3 only.
  4. Copy the key (starts with AIza) and paste it into TubeStack Setup Integrations or Settings → YouTube Data API key, then save.

4. OAuth consent screen (required before OAuth client)

  1. Go to APIs & Services → OAuth consent screen.
  2. Choose External (appropriate for personal use and testing).
  3. Fill in the required app name, user support email, and developer contact information.
  4. On Scopes, add the scope TubeStack requests: https://www.googleapis.com/auth/youtube (manage your YouTube account — needed for playlist creation and importing your playlists).
Testing mode — add yourself, do not publish. While the app stays in Testing, only accounts listed under Test users can sign in. Open Audience (or the test-users step on the consent screen), click Add users, and add the full Google email address you use in Chrome (the same account that should authorize TubeStack). You can add a small set of personal or family accounts the same way. Do not click Publish app unless you intend to go through Google’s production verification process for a public app. For personal TubeStack use, keeping the app in Testing with your email in Test users is normal and expected.

5. OAuth 2.0 Client ID (Web application)

  1. Still under Credentials, click Create credentials → OAuth client ID.
  2. Application type: Web application.
  3. Under Authorized redirect URIs, add exactly the redirect URI shown in TubeStack Settings → YouTube OAuth (it is a Chrome extension URL — copy from there so it matches your install).
  4. Create the client, then copy the Client ID (ends with .apps.googleusercontent.com) into TubeStack Settings → YouTube OAuth, and save.
  5. Use Test Google sign-in in Settings to confirm the flow completes.

If Google shows “access blocked” or “app not verified,” double-check that your Google account’s email is listed under Test users and that the OAuth client is Web with the correct redirect URI.

OpenAI API key (optional)

Some dashboard tools (for example AI-assisted category work) can call OpenAI over the network using an API key you store in TubeStack. That key is kept on this device and is only sent to OpenAI when you run those features.

  1. Sign in at platform.openai.com (this is the OpenAI API platform, not the same product as the consumer ChatGPT website).
  2. Open API keys and create a new secret key. Copy it once and paste it into TubeStack Settings → OpenAI API key, then save.
  3. OpenAI bills API usage from organization billing: add a payment method and buy prepaid credits (or otherwise enable billing as shown on that site) so requests can succeed. Pricing and credit packs change over time — use OpenAI’s billing page for current options.

ChatGPT Plus (or other ChatGPT subscriptions) is separate from API billing. Paying for ChatGPT does not automatically fund API calls; API usage is charged through the platform account and credits/billing you configure for API access.

Use Test API connection in Settings after saving the key to confirm the extension can reach OpenAI.
