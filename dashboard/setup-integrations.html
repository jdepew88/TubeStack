Local-first

TubeStack works locally without Google, YouTube API, OpenAI, or any TubeStack server. Optional integrations only run when you enable and use them.

Setup checklist

    YouTube Data API key

    Used for metadata, playlist imports, channel scans, and YouTube Data API features. Your API key is stored locally in Chrome extension storage. It is sent only to Google’s YouTube Data API when you use YouTube API features.

    Paste a key to add or replace.

    How to get a YouTube Data API key
    1. Go to Google Cloud Console.
    2. Create or select a project.
    3. Enable YouTube Data API v3.
    4. Create an API key under Credentials.
    5. Paste it into TubeStack and test.
    6. Optional: restrict the key to YouTube Data API v3.

    Google OAuth Client ID

    Used for user-authorized YouTube actions such as playlist import/export. TubeStack uses your own Google Cloud OAuth Client ID. OAuth access tokens stay temporary in this extension — TubeStack does not run a backend to store Google account tokens.

    How to get an OAuth Client ID
    1. Go to Google Cloud Console → OAuth consent screen and configure it.
    2. Create OAuth 2.0 Client ID → type Web application.
    3. Add the redirect URI shown above (from chrome.identity.getRedirectURL()).
    4. Paste the Client ID into TubeStack and use Test Google sign-in.

    OpenAI API key

    OpenAI is optional. TubeStack only contacts OpenAI when you explicitly use AI features. Selected video metadata or notes may be sent depending on the feature.

    Paste a key to add or replace.

    How to get an OpenAI API key
    1. Sign in at OpenAI.
    2. Create an API key under API keys.
    3. Paste it here and test when you want AI features.
